---
description: Markdown 문서를 원문 왜곡 없이 A4 Microsoft Word 문서(.docx)로 변환
argument-hint: "<input.md> [output.docx]"
---

/a4 또는 /skill:a4 요청으로 실행한다. 반드시 `a4` 스킬(`~/.agents/skills/a4/SKILL.md`)을 사용해 Markdown 문서를 원문 왜곡 없이 A4 Microsoft Word 문서(`.docx`)로 변환한다. HTML은 사용자가 명시적으로 HTML을 요구한 경우에만 생성한다.

입력 인자:
- `$1`: 변환할 Markdown 파일 경로
- `$2`: 선택 출력 Word DOCX 파일 경로
- `$ARGUMENTS`: 사용자가 추가로 적은 요구사항

작업 규칙:
1. `~/.agents/skills/a4/SKILL.md`를 읽고 지침을 따른다.
2. 원문 텍스트·순서·제목 계층·목록·표·숫자·날짜·이름을 수정하지 않는다.
3. 출력 경로가 제공되지 않으면 입력 파일명 기준 `.a4.docx`로 생성한다.
4. 기본 실행:
   - 출력 경로가 있을 때: `python3 ~/.agents/skills/a4/scripts/md-to-a4-docx.py "$1" -o "$2"`
   - 출력 경로가 없을 때: `python3 ~/.agents/skills/a4/scripts/md-to-a4-docx.py "$1"`
5. Word 문서는 Noto Sans CJK KR 폰트를 사용하고, `/Users/mindasom/dev/dotandco/doc/[최종본]도트앤코_계약서_수정본.docx` 같은 깔끔한 계약서/제안서 스타일을 따른다.
6. 생성 후 heading count check, list/heading formatting check, 주요 파일 경로를 보고한다.

사용자 추가 요청:
$ARGUMENTS
