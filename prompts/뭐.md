---
description: 지난 세션이나 최근 커밋 변경사항을 짧고 읽기 쉽게 요약
---
지금 작업 중인 프로젝트/폴더 기준으로 요약을 만들어줘.

우선순위:
1. 현재 작업 디렉터리와 같은 `cwd`를 가진 Pi 세션 파일을 `~/.pi/agent/sessions/` 아래에서 찾고, 가능하면 **현재 세션은 제외한 바로 이전 세션**을 골라 요약해.
2. 그런 세션이 없으면 Git의 **최근 10개 커밋 메시지와 diff**를 읽고 무엇이 바뀌었는지 요약해.

작성 방식:
- **비개발자도 바로 이해할 수 있게** 써.
- 무엇을 고쳤는지, 왜 바뀌었는지, 사용자 입장에서 뭐가 달라졌는지만 남기고 **짧게 압축**해.
- 내부 구현 설명, 파일명 나열, 코드 용어는 꼭 필요할 때만 최소화해.
- 추측하지 말고 확인한 사실만 써.

출력 형식:
- 첫 줄은 `[지난 세션 요약]` 또는 `[최근 커밋 요약]`으로 시작해.
- 그 아래는 **짧은 불릿 리스트 2~4개**로 정리해.
- 각 불릿은 **한 줄 또는 두 줄 안쪽**으로 짧게 써.
- **불릿과 불릿 사이에는 빈 줄을 1줄씩 넣어** 숨을 띄워줘.
- 핵심만 남기고, 비슷한 내용은 합쳐서 **압축적으로** 써.
- 문단형 줄글보다, 한눈에 훑히는 **목록형 요약**을 우선해.
- 불릿마다 가능하면 `무엇을 고쳤는지 / 왜 바뀌었는지 / 사용자에게 뭐가 달라졌는지`가 드러나게 써.
- 파일명 나열, 내부 구현 설명, 장황한 배경 설명은 피하고, 확인한 사실만 써.

예시 스타일:
[지난 세션 요약]
- 로그인 흐름이 중간에 끊기던 문제를 잡아서, 사용자가 다시 처음부터 입력하지 않도록 고쳤어요.

- 설정 화면을 정리해서, 자주 쓰는 옵션을 더 빨리 찾을 수 있게 바꿨어요.

- 전체적으로 헷갈리던 부분을 줄이고, 다시 시도하기 쉬운 상태로 다듬은 작업이었어요.

지난 세션도 없고 Git 기록도 충분하지 않으면, 그 사실을 첫 불릿에서 짧게 밝히고 확인 가능한 내용만 정리해.
